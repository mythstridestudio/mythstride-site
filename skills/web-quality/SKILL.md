# Web Quality Skill

## Goal
Ensure the MythStride website feels premium, fast, responsive, and production-ready.

## Checks before finishing

### Visual Quality
- Avoid generic SaaS UI.
- Avoid random gradients.
- Keep dark fantasy identity consistent.
- Use spacing, contrast, and hierarchy intentionally.
- Every section must have a clear purpose.

### UX Quality
- Mobile-first layout.
- Clear CTA above the fold.
- No clutter.
- No confusing navigation.
- Text must be readable on dark backgrounds.

### Motion Quality
- Animations must be subtle.
- Use Framer Motion for component entrance.
- Use GSAP only for parallax or scroll scenes.
- Respect reduced motion when possible.
- Avoid animation overload.

### Technical Quality
- Run lint.
- Run build.
- Remove unused components.
- Avoid hydration errors.
- Avoid heavy assets without optimization.

### Performance
- Optimize images.
- Prefer static rendering.
- Avoid unnecessary client components.
- Keep hero animations lightweight.
- Check Lighthouse when possible.

## Final output expected
Before finishing, report:
- files changed
- visual improvements
- UX improvements
- performance risks
- commands tested
